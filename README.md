# Zwillingstag – CDU/CSU Bundestag Digital Twin

A real-time simulation of how individual CDU/CSU members of the German Bundestag react to parliamentary debates. Each member's reaction (applause, remark, question) is generated by an LLM based on their political profile and the actual speech being delivered.

## Features

- 🏛️ **Semi-circular parliament seating layout** – 40 CDU/CSU members arranged as in the real Bundestag
- 👏 **Live reaction animations** – clapping, speech bubbles for remarks and questions
- 🤖 **LLM-powered reactions** – single OpenAI call per speech generates all 40 member reactions
- 💾 **Reaction caching** – speeches are only processed once; revisiting a speech is instant
- 📡 **WebSocket real-time updates** – reactions push to all connected browsers
- 🕐 **Timeline navigation** – browse past speeches and see how the faction reacted
- 📰 **Bundestag DIP API integration** – uses official open-data API for live speeches (mock fallback included)
- 🔌 **Works without API keys** – mock data lets you explore the UI immediately

## Architecture

```
backend/   (Python 3.12 / FastAPI)
  main.py                    ← FastAPI app, WebSocket endpoint, REST routes
  models.py                  ← Pydantic data models
  services/
    bundestag_api.py         ← Bundestag DIP API client (with mock fallback)
    llm_service.py           ← OpenAI reaction generation (with mock fallback)
    debate_simulator.py      ← Orchestration: seat layout, caching, live updates
  data/
    cdu_members.json         ← 40 CDU/CSU member profiles

frontend/  (React 18 / Vite)
  src/
    App.jsx                  ← Root component
    hooks/useSimulation.js   ← WebSocket client hook
    components/
      ParliamentView.jsx     ← SVG semi-circular seat layout
      MemberSeat.jsx         ← Individual seat with reactions & tooltip
      DebatePanel.jsx        ← Current speech display
      TimelineControl.jsx    ← Speech history navigation
      ReactionLegend.jsx     ← Icon legend
    styles/index.css         ← Dark government-style theme
```

## Quick Start

### Backend

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # optionally add API keys
uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev                   # starts on http://localhost:5173
```

> Without API keys the backend uses 5 built-in mock speeches and generates plausible (seeded-random) reactions. Add `OPENAI_API_KEY` for LLM-powered reactions.

---

## Configuration (backend/.env)

| Variable | Description | Default |
|---|---|---|
| `BUNDESTAG_API_KEY` | [DIP API key](https://dip.bundestag.de/%C3%BCber-dip/hilfe/api) | *(mock fallback)* |
| `OPENAI_API_KEY` | OpenAI key for reaction generation | *(mock fallback)* |
| `OPENAI_MODEL` | Model to use | `gpt-4o-mini` |
| `POLL_INTERVAL_SECONDS` | Live-update polling interval | `120` |

## API

| Endpoint | Method | Description |
|---|---|---|
| `/api/health` | GET | Health check |
| `/api/members` | GET | All 40 CDU/CSU members with seat positions |
| `/api/speeches` | GET | Available recent speeches |
| `/api/reactions/{speech_id}` | GET | Reactions for a specific speech |
| `/api/state` | GET | Full simulation state (speech + reactions + timeline) |
| `/ws` | WS | Real-time state updates; accepts `select_speech` / `refresh` messages |

Swagger UI: http://localhost:8000/docs
